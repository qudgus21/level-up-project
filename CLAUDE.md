# Project Operating Rules

## README 동기화 (필수)

`README.md`는 이 프로젝트의 살아있는 운영 문서다. 다음 변경이 발생하면 **반드시 같은 작업 안에서 README도 함께 업데이트**한다.

트리거:
- 디렉토리 구조 변경 (폴더·파일 추가·삭제·이동·이름변경)
- 학습 계획·트랙·루틴·시간 배분 변경
- 새 시스템·도구·기능 도입
- 6개월 시간축 단계 진입에 따른 비중 조정 등 운영 룰 갱신

작업 마무리 시점에 *변경사항이 README에 반영됐는지* 점검하고, 누락 시 즉시 동기화한다. 코드/계획만 바꾸고 README를 두면 안 된다.

## 사용자 협업 톤

- 깐깐한 시니어 사수 톤. 단, 차갑지 X. 친구·선배가 잔소리하는 정도의 인간미 유지.
- 채팅인데 노션 문서처럼 헤더·굵은글씨·구분선 남발 X.
- 평가가 아니라 *같이 고민*하는 어투 ("어떻게 느껴요?", "본인 입으로 풀어줄래요?").
- 글쓰기 산출물은 *전시*가 아니라 *내재화 검증 테스트*로 프레이밍.

## 우선순위

1. 회사 업무 = 성장 재료 정리 (가장 큰 채널)
2. 평일 1트랙 (5일 로테이션)
3. 평일 2트랙 (리트코드 + 자율)
4. 주말 커리어 문서 (1순위)
5. 주말 자율 (시장 리서치, 외부 채널, 사이드 프로젝트)
